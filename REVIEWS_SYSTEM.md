# Система отзывов FreelanceHub

## Описание

Система позволяет пользователям оставлять отзывы о работе бота. За каждый одобренный отзыв пользователь получает **3 дня x2 статуса**.

## Архитектура

### 1. База данных

**Таблица `reviews`:**
- `id` - ID отзыва
- `tg_id` - Telegram ID пользователя
- `username`, `first_name` - Данные пользователя
- `text` - Текст отзыва
- `rating` - Оценка от 1 до 5 (опционально)
- `status` - Статус: `pending`, `approved`, `rejected`
- `moderator_id` - ID модератора
- `moderated_at` - Дата модерации
- `rejection_reason` - Причина отклонения
- `x2_awarded` - Было ли начислено x2
- `x2_awarded_at` - Когда начислено x2
- `created_at` - Дата создания

### 2. Бот для отзывов (`feedback_bot/`)

**Расположение:** `kote/feedback_bot/`

**Функционал:**
- Прием отзывов от пользователей через команду `/review`
- Сохранение отзывов в БД со статусом `pending`
- Уведомление пользователей о статусе модерации

**Команды:**
- `/start` - Приветственное сообщение
- `/review` - Оставить отзыв
- `/help` - Справка

### 3. Модерация в основном боте

**Расположение:** `vacancy_bot/handlers/admin.py`

**Функционал:**
- Просмотр отзывов на модерацию через админ-панель
- Одобрение отзывов (начисление 3 дней x2)
- Отклонение отзывов с указанием причины
- Уведомление пользователей о результате модерации

**Доступ:** Админ-панель → "Модерация отзывов"

## Установка

### 1. Миграция БД

Выполните миграцию в Supabase SQL Editor:
```sql
-- Файл: vacancy_bot/database/migration_add_reviews.sql
```

### 2. Настройка бота отзывов

1. Создайте бота через @BotFather
2. Скопируйте `.env.example` в `.env` в папке `feedback_bot/`
3. Заполните `FEEDBACK_BOT_TOKEN`
4. Укажите те же `SUPABASE_URL` и `SUPABASE_KEY` что и в основном боте

### 3. Настройка основного бота

В `.env` основного бота добавьте:
```
FEEDBACK_BOT=@your_feedback_bot
```

### 4. Запуск

**Бот отзывов:**
```bash
cd feedback_bot
python main.py
```

**Основной бот:**
```bash
cd vacancy_bot
python main.py
```

## Процесс работы

1. **Пользователь оставляет отзыв:**
   - Переходит в бот отзывов
   - Использует команду `/review`
   - Вводит текст отзыва и опционально оценку
   - Отзыв сохраняется со статусом `pending`

2. **Модерация:**
   - Администратор заходит в админ-панель основного бота
   - Выбирает "Модерация отзывов"
   - Просматривает отзывы и принимает решение:
     - ✅ **Одобрить** - начисляется 3 дня x2, пользователь получает уведомление
     - ❌ **Отклонить** - указывается причина, пользователь получает уведомление

3. **Начисление x2:**
   - При одобрении отзыва автоматически вызывается `award_x2_status(user_tg_id, days=3)`
   - Если у пользователя уже есть x2 статус - он продлевается
   - Если нет - создается новый сроком на 3 дня

## Интеграция с закрепленным сообщением

В закрепленном сообщении о реферальной системе добавлена информация о системе отзывов:
- Ссылка на бота отзывов
- Информация о награде (3 дня x2 за одобренный отзыв)
